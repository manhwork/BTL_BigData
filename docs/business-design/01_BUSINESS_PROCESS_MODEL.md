## 1. TỔNG QUAN QUY TRÌNH NGHIỆP VỤ

### 1.1. Sơ đồ nghiệp vụ tổng thể (High-Level BPMN)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           HỆ THỐNG SO SÁNH GIÁ                              │
├─────────────────────────────────────────────────────────────────────────────┤

│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌───────────┐ │
│  │   DATA       │    │    DATA      │    │   USER       │    │  ALERT    │ │
│  │   CRAWLING   │───▶│  PROCESSING  │───▶│  SERVICES    │───▶│  SYSTEM   │ │
│  │   LAYER      │    │    LAYER     │    │   LAYER      │    │           │ │
│  └──────────────┘    └──────────────┘    └──────────────┘    └───────────┘ │
│         │                   │                   │                  │        │

│  ┌──────────────────────────────────────────────────────────────────────┐  │

│  │         (PostgreSQL + MongoDB + Redis + Elasticsearch)                │  │
│  └──────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2. Các quy trình nghiệp vụ chính

| ID | Quy trình | Mô tả | Actor chính |
|----|-----------|-------|-------------|
| BP-01 | Thu thập dữ liệu sản phẩm | Crawl dữ liệu từ các sàn TMĐT | System (Crawler) |
| BP-02 | Phân loại Shop | Tính điểm và phân loại độ tin cậy shop | System (Processor) |
| BP-03 | So sánh giá sản phẩm | Matching và so sánh giá giữa các sàn | User + System |
| BP-04 | Theo dõi biến động giá | Lưu trữ và phân tích lịch sử giá | System |
| BP-05 | Cảnh báo giá | Gửi thông báo khi đạt điều kiện | System (Alert) |
| BP-06 | Quản lý người dùng | Đăng ký, đăng nhập, quản lý profile | User |
| BP-07 | Quản lý Watchlist | Theo dõi sản phẩm yêu thích | User |

#### 2.1.2. Mô tả chi tiết các bước

| Step | Tên bước | Mô tả | Input | Output | Actor |
|------|----------|-------|-------|--------|-------|
| 1 | Timer Trigger | Scheduler kích hoạt theo lịch | Cron expression | Job event | Scheduler |
| 2 | Kiểm tra lịch | Xác định loại sản phẩm cần crawl | Current time | Product list | Scheduler |
| 3 | Tạo Job Queue | Đưa job vào hàng đợi | Product list | Job ID | Queue Manager |
| 4 | Worker xử lý | Lấy job từ queue và thực thi | Job ID | Raw data | Crawler Worker |
| 5 | Crawl dữ liệu | Thu thập từ sàn TMĐT | URL, Selectors | HTML/JSON | Crawler |
| 6 | Validate | Kiểm tra tính hợp lệ | Raw data | Validated data | Validator |
| 7 | Chuẩn hóa | ETL process | Validated data | Normalized data | ETL Pipeline |
| 8 | Lưu trữ | Insert/Update database | Normalized data | DB records | Data Layer |
| 9 | Alert Check | Kiểm tra điều kiện cảnh báo | New price | Alert events | Alert Service |

#### 2.3.2. Thuật toán Product Matching

```
┌─────────────────────────────────────────────────────────────────┐
│                    PRODUCT MATCHING ALGORITHM                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Input: Product Name/URL                                         │
│                                                                  │
│  Step 1: Exact Matching                                          │
│  ├── So sánh tên chính xác                                       │
│  ├── So sánh mã sản phẩm (SKU, Barcode)                         │
│  └── Match ≥ 95% → Cùng sản phẩm                                │
│                                                                  │
│  Step 2: Fuzzy Matching (nếu Step 1 không match)                │
│  ├── Levenshtein Distance                                        │
│  ├── Cosine Similarity (TF-IDF)                                 │
│  └── Match ≥ 85% → Cùng sản phẩm                                │
│                                                                  │
│  Step 3: Attribute Matching                                      │
│  ├── So sánh: Brand, Color, Size, Model                         │
│  ├── Weighted scoring                                            │
│  └── Match ≥ 70% → Sản phẩm tương tự                            │
│                                                                  │
│  Step 4: Image Matching (Optional)                               │
│  ├── Feature extraction (CNN)                                    │
│  ├── Visual similarity                                           │
│  └── Match ≥ 80% → Cùng sản phẩm                                │
│                                                                  │
│  Output: Matched Products List với Confidence Score              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.5.2. Các loại Alert Condition

| Alert Type | Điều kiện | Ví dụ |
|------------|-----------|-------|
| Price Drop | `current_price < target_price` | Giá giảm xuống dưới 1,000,000đ |
| Percentage Drop | `discount_pct >= target_pct` | Giảm giá >= 30% |
| Best Price | `current_price <= min_price_30d` | Giá tốt nhất 30 ngày |
| Flash Sale | `event_tag = 'flash_sale'` | Có flash sale |
| Cross Platform | `price_A < price_B * threshold` | Shopee rẻ hơn Lazada 10% |
| Stock Low | `stock_quantity < threshold` | Còn dưới 10 sản phẩm |
